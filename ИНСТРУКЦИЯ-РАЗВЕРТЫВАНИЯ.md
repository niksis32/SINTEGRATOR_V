# Инструкция по проекту `sintegrator-site_v`

Краткая и достаточно подробная памятка: что это за приложение, что в нём сделано, как перенести в любую папку и запустить без остальных проектов в workspace.

---

## Кратко: что это

Одностраничный лендинг **«Синтегратор»** на **React 19 + Vite 7**. Статический фронтенд (после сборки — обычные HTML/CSS/JS).

Визуальная система и наполнение завязаны на экспорт из папки **`stitch_sintegrator_enterprise_digital_architecture/`** (Stitch/Google AI Studio):

- светлая палитра, «machined steel / editorial»
- шрифты **Space Grotesk** и **Inter** (Google Fonts)
- иконки **Material Symbols Outlined**
- блоки: шапка, главный экран, отрасли («протокол отраслей»), услуги и схема решений, CTA, кейсы, отзывы, «о компании» (герой + таймлайн + ценности), второй CTA, контакты с картой, подвал

Исходники макетов Stitch **не подключаются во время сборки** — они только эталон; верстка и стили перенесены в React и `src/styles.css`.

---

## Структура каталогов (важное для переноса)

Переносите **целиком** папку проекта (например `sintegrator-site_v`). Достаточно следующего минимума:

| Путь | Назначение |
|------|------------|
| `package.json` | зависимости и скрипты npm |
| `package-lock.json` | фиксируемые версии пакетов *(рекомендуется переносить)* |
| `vite.config.js` | конфигурация Vite и React-плагина |
| `index.html` | точка входа HTML, мета-теги, подключение шрифтов |
| `src/` | весь код приложения |
| `src/main.jsx`, `src/App.jsx` | точка входа JS и сборка секций |
| `src/styles.css` | общие стили Stitch |
| `src/data/siteData.js` | тексты, ссылки на изображения, навигация |
| `src/components/` | секции страницы и мелкие UI-части |
| `public/` | статика Vite *(если есть — переносите)* |
| `stitch_sintegrator_enterprise_digital_architecture/` | **опционально**, только как справочник макетов, для работы сайта **не нужна** |

Папки **`node_modules/`** и **`dist/`** при переносе **не обязательны** — их можно не копировать; их восстанавливают командами ниже.

## Требования к окружению

- **Node.js** актуальной LTS (подойдёт ветка 20+ или как у вас уже стоит для других Vite-проектов).
- **npm** (идёт вместе с Node).

Проверка:

```bash
node -v
npm -v
```

## Перенос в другую папку

1. Скопируйте всю папку проекта (или заархивируйте и распакуйте целевым именем).
2. Убедитесь, что внутри есть `package.json` и каталог `src/`.
3. Откройте терминал и перейдите **в корень этой папки** (ту, где лежит `package.json`).

Пример (Windows, PowerShell):

```powershell
cd D:\путь\к\sintegrator-site_v
```

Пример (cmd):

```cmd
cd /d D:\путь\к\sintegrator-site_v
```

Если команда `npm` в PowerShell даёт ошибку политики выполнения для `npm.ps1`, используйте:

```powershell
npm.cmd install
npm.cmd run dev
```

либо временно для сессии:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## Установка зависимостей

Один раз в корне проекта:

```bash
npm install
```

Будут созданы `node_modules/` по `package.json` / `package-lock.json`.

## Локальный запуск (разработка)

```bash
npm run dev
```

По умолчанию Vite поднимает **`http://localhost:5173`**. Скрипт с `--host 0.0.0.0` также открывает доступ по IP в локальной сети.

Если порт **5173** занят другим проектом:

```bash
npm run dev -- --port 5174
```

Откройте в браузере указанный адрес (например `http://localhost:5174`).

## Production-сборка

```bash
npm run build
```

Результат в каталоге **`dist/`**:

- статические файлы, готовые к выкладке на любой статический хостинг (nginx, Apache, CDN, GitHub Pages, облако и т.д.);
- содержимое **`dist/`** нужно опубликовать **как корень сайта**.

Просмотр собранной версии локально:

```bash
npm run preview
```

---

## Выкладка на хостинг (общие шаги)

1. На машине с Node выполните `npm install` и `npm run build`.
2. Загрузите **содержимое** папки `dist/` в корень веб-сайта *(или укажите в панели хостинга каталог сборки как docroot)*.

Если сайт должен жить **не в корне домена**, а в подкаталоге (например `https://example.com/site/`), в `vite.config.js` нужно задать `base`, например `base: '/site/'`, затем снова выполнить `npm run build`. Под свой URL уточните у администратора хостинга.

Формы на странице сейчас **демонстрационные** (нет отправки на сервер): перед бой подключите backend, webhook (например Bitrix24) или почтовый сервис.

## Где править контент и внешний вид

| Задача | Где править |
|--------|--------------|
| Тексты, телефоны, email, блоки кейсов и т.д. | `src/data/siteData.js` |
| Порядок секций на странице | `src/App.jsx` |
| Разметка блоков и классы секций | `src/components/*.jsx` |
| Цвета, сетки, типографика Stitch | `src/styles.css` |
| `<title>`, описание, шрифты в `<head>` | `index.html` |

Эталонные HTML из Stitch лежат в **`stitch_sintegrator_enterprise_digital_architecture/`** — при изменении макета удобно сверять с ними глазами; в прод-сборке эта папка не участвует.

## Типичные проблемы

| Симптом | Что проверить |
|---------|----------------|
| `Could not read package.json` | Терминал открыт не в папке с `package.json` — выполните `cd` в корень проекта. |
| PowerShell блокирует `npm` | `npm.cmd` или `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`. |
| Порт занят | `npm run dev -- --port НОМЕР`. |
| Пустая страница на хостинге | Залит не `dist/`, или не в корень, или нужен другой `base` в Vite для подпапки. |

## Версионирование

В проекте зафиксированы версии зависимостей в **`package-lock.json`**. После переноса выполняйте `npm install` в новой локации — lock-файл воспроизведёт те же версии пакетов при неизменённом файле.
